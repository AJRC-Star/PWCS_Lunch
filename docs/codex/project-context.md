# Project Context

Durable, shared project context for Codex and collaborators.

## Overview

PWCS Lunch is a mobile-first React/Vite app that displays the Benton Middle School lunch menu with a native-app feel. Production is served from GitHub Pages at `https://ajrc-star.github.io/PWCS_Lunch/`.

The app does not call MealViewer directly in normal production use. It reads the committed `public/menu-data.json` artifact, which is generated by `scripts/fetch-menu.ts` and validated before publication.

## Architecture

The app shell in `src/` loads a normalized menu snapshot, manages browser cache behavior, and renders full-screen day navigation and menu cards. Shared meal normalization, contracts, artifact validation assumptions, and school-calendar helpers live in `shared/` so the browser app and Node scripts use the same rules.

Menu refresh is intentionally local-network based: MealViewer blocks GitHub-hosted runner IPs, so the scheduled GitHub Actions fetch is disabled and a Codex local automation runs the established refresh script instead.

## Important Paths

| Path | Purpose |
|---|---|
| `src/` | React app, UI components, styles, cache behavior, and app tests. |
| `src/components/` | Day tabs, day card, and skeleton loading components. |
| `shared/` | Menu contracts, normalization logic, calendar helpers, and shared regression tests. |
| `scripts/fetch-menu.ts` | Fetches MealViewer data and writes the normalized artifact. |
| `scripts/local-fetch.sh` | Local weekly refresh wrapper used by Codex automation. |
| `scripts/validate-menu-artifact.ts` | Validates the committed artifact against runtime rules. |
| `public/menu-data.json` | Published menu snapshot served by the GitHub Pages app. |
| `.github/workflows/` | CI, deploy, freshness checks, and manual menu fetch workflow. |
| `docs/codex/` | Checked-in Codex/project operating docs. |

## Conventions

- Keep the UI mobile-first and full-screen, with no page scrolling in the core app experience.
- Preserve the single-source-of-truth model: the published weekly artifact is the authoritative menu source.
- Reuse shared normalization and contract logic instead of duplicating menu rules in app or script code.
- Keep changes focused and reviewable; avoid broad rewrites or unrelated cleanup.
- Treat school, student, and family-adjacent data as sensitive.

## Constraints and Non-Goals

- Use Node 22 locally to match GitHub Actions.
- GitHub Pages deployment uses the `/PWCS_Lunch/` base path unless `VITE_BASE_PATH` overrides it.
- MealViewer requests should not depend on GitHub-hosted runners for the normal refresh path.
- Do not commit secrets, private credentials, raw student data, or personal records.
- Do not add a competing root instruction file such as `Codex.md`.
- Private cross-thread memory belongs in Codex native Memories, not checked-in docs.

## External Systems

- MealViewer public API: `https://api.mealviewer.com/api/v4/school/BENTONMIDDLE/{startDate}/{endDate}`.
- GitHub Actions runs CI, deployment, freshness checks, and manual fetch dispatches.
- GitHub Pages hosts production.
- Codex automation `weekly-pwcs-lunch-menu-refresh` runs the local refresh workflow Saturdays at 06:00 America/New_York.
